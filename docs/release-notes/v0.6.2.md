# v0.6.2 Release Notes

## Purpose

Harden the planning side of the harness after gaps were found in expectation derivation, expectation validation, task derivation, and task validation.

## Added

- `expectation-derivation-agent`
- `expectation-validation-agent`
- `task-decomposition-agent`
- `task-contract-validation-agent`
- `validate_expectations.py`
- `validate_tasks.py`
- `simulate_pipeline_workflow.py`
- planning result schemas
- top-level epic requirement in task contracts
- `/validate-tasks` command

## Fixed

- `derive-expectations` now requires schema and semantic validation.
- `validate-expectations` now produces a structured result.
- `derive-tasks` now explicitly uses `epic-task-decomposition` and requires top-level epics.
- Task contracts now fail validation when epics or test hierarchy declarations are missing.
