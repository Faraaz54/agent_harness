# v0.6.3 Release Notes

## Theme

Project-pack context is now first-class and schema-enforced across planning and downstream build phases.

## Added

- `scripts/project_context.py`
- `scripts/assemble_context.py`
- `schemas/project-context-index.schema.json`
- hardened `schemas/context-pack.schema.json`
- project-pack `context/` folders for `generic-python` and `invoice-governance`
- project context propagation into Expectations and task contracts
- context-aware action packets for implementation tasks
- project-context E2E simulation in `scripts/simulate_pipeline_workflow.py`

## Changed

- `/assemble-context` now runs deterministic scripts and validates output.
- `/derive-expectations` must include `source_context`.
- `/validate-expectations` checks context references.
- `/derive-tasks` must include task-level `required_context` and `context_files`.
- `/validate-tasks` checks task-level context references.
- `prepare_run.py` validates context, expectations, task contracts, expectation validation, and task validation before creating run artifacts.
