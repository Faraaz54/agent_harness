# v0.6.1 Release Notes

## Theme

Structured agent outputs and deterministic result validation.

## Why this release exists

The harness now requires any agent output consumed by later automation to be structured JSON. Prose-only reviews are not valid evidence for state transitions.

## Added

- `schemas/domain-review-result.schema.json`
- stricter `schemas/implementation-result.schema.json`
- stricter `schemas/review-result.schema.json`
- stricter `schemas/task-validation.schema.json`
- `scripts/agent_result_contracts.py`
- `docs/harness/agent-output-contracts.md`
- harness tests for structured agent result validation

## Changed

- `build-agent` now includes an exact required implementation-result JSON shape.
- `test-agent` now includes an exact required review-result JSON shape.
- `principal-engineer-agent` now includes an exact required review-result JSON shape.
- `domain-reviewer-agent` now includes an exact required domain-review JSON shape.
- `validator-agent` now includes an exact required task-validation JSON shape.
- `build_orchestrator.py record-result` validates output schemas and checks task/agent consistency before mutating run state.
- Project-pack domain review schemas were aligned with the core structured domain review contract.
- Operator manual now references structured agent output validation.

## Validation

```bash
python -B scripts/check_python_syntax.py scripts harness_tests
python -B -m pytest harness_tests -q
python -B scripts/validate_harness.py .

python -B scripts/project_pack.py validate --pack project-packs/invoice-governance
python -B scripts/verify_installation.py
python -B scripts/harness_selftest.py
```
