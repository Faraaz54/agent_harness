# v0.6.4 Release Notes

## Added

- First-class IDSD Technical Spec stage.
- `/derive-technical-spec` command.
- `/validate-technical-spec` command.
- `technical-spec-agent` and `technical-spec-validation-agent`.
- `technical-specification` skill.
- Technical Spec schemas and validator script.
- Task-contract schema now requires top-level `technical_spec` and per-task `technical_spec_refs`.
- `prepare_run.py` now requires validated Technical Spec artifacts.
- Build action context packets now include relevant Technical Spec sections for implementation/review/validation.
- Simulated pipeline workflow now exercises Technical Spec generation/validation before task decomposition.

## Intent

Prevent the build agent from inventing major architectural, dependency, module-boundary, or pipeline-design decisions during implementation.
