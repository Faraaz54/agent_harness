# v0.6.5 Release Notes

## Summary

v0.6.5 hardens `/build-next` and `/build-auto` so they no longer depend on a manual `build_orchestrator.py next` step.

## Added

- `scripts/build_driver.py`
- `scripts/build_next.py`
- `scripts/build_auto.py`
- `schemas/postflight-result.schema.json`
- `schemas/commit-result.schema.json`
- `docs/harness/build-next-auto-v0-6-5.md`

## Changed

- `/build-next` now starts by running `python -B scripts/build_next.py`.
- `/build-auto` now starts by running `python -B scripts/build_auto.py`.
- `scripts/postflight.py` now writes `docs/postflight/<task-id>.json`.
- `scripts/build_orchestrator.py record-result` now validates and records `RUN_POSTFLIGHT` and `COMMIT_TASK` results.
- `RUN_POSTFLIGHT` actions now declare their expected output path.

## Operator rule

Do not manually run `python -B scripts/build_orchestrator.py next` during normal `/build-next` or `/build-auto` execution. Use it only for debugging.
